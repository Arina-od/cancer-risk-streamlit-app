
# Cancer Risk Streamlit App

Готовый проект для GitHub: обучение пяти моделей риска и русскоязычное Streamlit-приложение.

## Что внутри

- `train_models.py` — отдельный скрипт для обучения и сохранения моделей.
- `app.py` — отдельное Streamlit-приложение.
- `cancer_risk_config.py` — единая конфигурация: имена признаков, маппинг названий колонок, список типов рака.
- `models/` — сюда сохраняются `.joblib`-артефакты.
- `reports/` — сюда сохраняются сводные метрики после обучения.
- `.streamlit/config.toml` — тема приложения.
- `requirements.txt` — зависимости для локального запуска и деплоя.

## Что я привёл к единому виду

В ноутбуке признаки и служебные поля были в `Camel_Case` / `Title_Case` формате:

- `Age`
- `BMI`
- `Family_History`
- `BRCA_Mutation`
- `H_Pylori_Infection`
- `Overall_Risk_Score`
- `Cancer_Type`

В проекте для GitHub они нормализованы в единый `snake_case` формат:

- `age`
- `bmi`
- `family_history`
- `brca_mutation`
- `h_pylori_infection`
- `overall_risk_score`
- `cancer_type`

Это сделано специально, чтобы:
- приложение было легче поддерживать,
- признаки в коде Streamlit и в коде обучения совпадали,
- не было путаницы между названиями из датасета и названиями в интерфейсе.

## Какие типы рака используются

В проекте используются только разделы из ноутбука:

- Breast
- Prostate
- Skin
- Colon
- Lung

Раздел **«Машинное обучение: предсказание вида рака»** намеренно не используется.

## Важная логика проекта

Каждый тип рака обучается как **отдельная регрессионная модель**, которая предсказывает:
- `overall_risk_score`

То есть приложение показывает не диагноз и не вероятность наличия рака, а прогнозируемую **оценку риска** на основе датасета проекта.

## Как обучить модели

Скрипт `train_models.py`:
- загружает исходный датасет по той же ссылке, что использовалась в ноутбуке,
- переименовывает колонки,
- для каждого типа рака делает отдельный train/test split,
- перебирает модели и гиперпараметры,
- выбирает лучшую модель по CV RMSE,
- сохраняет артефакты в `models/`.

### Запуск локально
```bash
pip install -r requirements.txt
python train_models.py
```

После запуска появятся:
- `models/breast_risk_model.joblib`
- `models/prostate_risk_model.joblib`
- `models/skin_risk_model.joblib`
- `models/colon_risk_model.joblib`
- `models/lung_risk_model.joblib`

А также:
- `reports/model_summary.csv`
- `reports/model_summary.json`

## Как запустить приложение локально

После обучения моделей:

```bash
streamlit run app.py
```

## Как выложить в GitHub

1. Создай новый репозиторий на GitHub.
2. Положи в него все файлы проекта.
3. **Обязательно** добавь уже обученные файлы из папки `models/`, иначе приложение в облаке не сможет стартовать.
4. Сделай commit и push.

## Почему в приложении есть проценты

Модели из ноутбука предсказывают `overall_risk_score`, который в датасете выглядит как число от 0 до 1. В интерфейсе это число дополнительно переводится в проценты для удобства чтения.

Пример:
- `0.42` → `42%`

## Дисклеймер

Приложение носит демонстрационный и учебный характер. Оно:
- не ставит диагноз,
- не является медицинским заключением,
- не заменяет консультацию врача.

## Рекомендуемый порядок работы

1. Скопируй этот проект в отдельный репозиторий.
2. Запусти `train_models.py` в Google Colab или локально.
3. Убедись, что папка `models/` заполнилась.
4. Закоммить обученные артефакты.
5. Задеплой `app.py` в Streamlit Community Cloud.
